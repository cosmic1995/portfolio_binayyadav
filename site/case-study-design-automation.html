Overview


The creative production team at Lowe's India spent significant hours each week manually producing and quality-checking ad creatives in Figma.

The process was repetitive and error-prone — the same formatting mistakes reappeared across batches, and each one required manual correction before work could progress.

I identified the problem, proposed the solution, and built it.

The Challenge

High-volume production Error-prone manual workflow Recurring QA overhead

Creative ad production at scale creates a specific category of friction: the work is largely systematic, but it is executed manually. Templates drift from brand guidelines. Sizes get misapplied. Typography rules break. Spacing becomes inconsistent. The opportunity was not to redesign the creative output — it was to automate the parts of the process that should never require a designer's attention in the first place.

Research & Discovery

I mapped the existing production process end to end — conducting internal interviews with the production team to understand where errors most frequently originated, which steps consumed the most time, and which tasks were rule-based enough to be codified reliably.

The research produced a clear picture: a significant portion of weekly production hours were spent on tasks that followed fixed, repeatable rules. Every one of those hours was an opportunity to automate — and a risk, because manual execution of rule-based tasks is where errors enter the system.


What I Built

A custom Figma automation plugin for creative ad design production. The plugin automates the repetitive production steps and runs quality validation checks before export — catching errors at the source rather than at the review stage.

The plugin handles: automated layout application across ad size variants; brand compliance validation covering colour, typography, and spacing rules; quality checks that catch recurring errors automatically; and batch processing of multiple creatives in a single run.


The Build Process

Designing the plugin required treating the production team as the user. The same research and empathy applied to any UX project — understanding what someone needs to do, where the current process fails them, and removing that friction — applied here.

I designed the plugin's interface as a UX problem first. The technical implementation used the Figma Plugin API and JavaScript. I built iteratively, testing each function with the production team and refining based on real usage before extending to the next feature.


The Result

45% reduction in production time. Recurring errors were eliminated entirely. The team redirected hours previously spent on repetitive production and error correction toward higher-value design work.

The plugin was adopted as part of the standard production workflow at Lowe's India, directly supporting the team's broader digital transformation goals.


Tools & Methods

Figma Plugin API · JavaScript · Figma · Internal workflow analysis · User interviews with production team · Iterative testing
