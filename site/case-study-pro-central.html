Overview


Pro Central is a unified platform for Lowe's Pro customers — professional contractors who manage multiple active projects and rely on Lowe's as their primary materials partner.

The platform consolidates project management, inventory tracking, invoicing, and order management into a single experience, designed around how contractors actually work.


One platform for project management, reordering, invoicing, and inventory — designed for how contractors work.

The Problem

Pro customers are Lowe's highest-value segment. But the digital experience was not built for them — it was built for a consumer making a single purchase.

A contractor running an active project needs to track materials per job, reorder stock, generate invoices from Lowe's orders, and verify availability before committing to a project timeline. Before Pro Central, none of these were served by a connected experience.

The friction compounded across every project and every week. For a contractor managing multiple active jobs simultaneously, this was a recurring operational cost.


Disconnected tools, re-entered data, wasted time — every day, across every active project.

Research

Reordering: most frequent, most friction-heavy Invoice management outside the platform Inventory visibility at planning stage

Interviews and journey mapping with professional contractors revealed a consistent finding: every purchasing decision is made in the context of a named job with a deadline. The platform's category-based structure was the wrong model entirely for a user who arrives with a project in mind, not a search query.

The Solution

Project Dashboard Reorder Flow Integrated Invoicing Inventory Visibility

Pro Central is organised around projects, not departments. The home view shows every active job — status, materials ordered, outstanding items, budget tracking, and invoice state — in a single screen.


Project-first architecture: every tool a contractor needs, surfaced in the context of the job they're working on.

Reorder Flow

The most frequent Pro action, given the most direct path.

Materials ordered for each project are stored and accessible from the project view. Current stock levels are shown inline — no context-switching, no searching the product catalogue.

Integrated Invoicing

Invoice generation rebuilt within the project context.

Line items auto-populate from order history. The goal was to make the external tool contractors had been using redundant — by providing something that worked as well with no context-switching required.

Handoff & Reflection


All components built on Backyard Design System 4.

Handoff documentation included annotated Figma frames for all primary user flows, error and empty states, interaction specifications, and component-level notes mapping each element to its DS4 token.


Pro Central required designing for a user under genuine professional pressure. Every screen had to be evaluated against one question: is the most important action the most visible one?


Annotated Figma handoff

Full documentation for all flows, states, interactions, and DS4 token mapping.

Professional context

Designed for users under time pressure with real consequences for delays.

System coherence

Decisions across dashboard, reorder, and invoicing were held consistent throughout.
